## 体验产品

### 简介

apichain是一款按版本迭代和项目视角管理接口文档和自动化接口测试用例的工具，具有以下功能：

1. apichain按版本迭代管理接口文档，并在迭代上线后将版本迭代的接口文档按不同微服务合并到项目中

2. 在版本迭代内编写无需修改数据，对一系列接口串联调用，可反复执行的接口自动化测试用例

3. 将迭代的自动化测试用例导出到项目，用于对项目的回归测试

### 本文档目标用户

- 您是后端、前端、测试工程师，需要一款管理接口文档和接口测试用例的工具

- 对apichain这款软件的功能感兴趣，想要体验该产品

### 体验之旅

#### 准备一个mysql数据库

您需要准备一个mysql8数据库，如果没有，可参照使用如下脚本启动一个mysql服务。（生产环境使用，请注意mysql配置文件管理、日志文件存储、数据库持久化、账户权限隔离等）

```shell
docker pull mysql:8.0

docker run --name mysql-container -p 33088:3306 -e MYSQL_ROOT_PASSWORD=112233 -d mysql:8.0
```

拥有mysql数据库后，你就可以提供以下环境变量

- DB_HOST

  数据库主机地址，示例中是 192.168.1.5

- DB_PORT

  数据库端口号，示例中是 33088

- DB_USER

  数据库用户名，示例中是 root

- DB_PASS

  数据库密码，示例中是 112233

- DB_NAME

  数据库名称，可以是任意字符串，示例中是 apichain

⚠️注意：该数据库管理账号，需要对该 数据库 具有 建表、修改表结构、数据增删改查 权限

#### 导入测试数据

确保您准备的数据库已存在，如果还没有，请自行创建数据库。

如果连接数据库过程中报错：Public Key Retrieval is not allowed，需要在驱动属性中，将 allowPublicKeyRetrieval 修改为 true

执行以下sql语句新建数据库:`create database apichain;`

从[这个链接](https://raw.githubusercontent.com/jiangliuer326442/ApiChain/refs/heads/main/db/dump-apichain.sql) 下载测试数据，导入到您准备好的数据库中

#### 运行runner

基于您已经准备好的数据库连接信息，对以下shell命令修改一下数据库连接参数，即可启动一个本地运行的runner

```shell
docker volume create apichain_cache_data;

docker pull registry.cn-shanghai.aliyuncs.com/apichain/runner:1.2.4;

docker run -d 
-p 6588:6588 
-e DB_HOST=192.168.1.5
-e DB_PORT=33088
-e DB_USER=root
-e DB_PASS=112233
-e DB_NAME=apichain
-e DEPLOY_COUNTRY=CN
-e APICHAIN_SUPER_UID=44470bb9b4f8d601f812945fe275e139
-v apichain_cache_data:/opt/cache
--name apichain-runner
registry.cn-shanghai.aliyuncs.com/apichain/runner:1.2.4
```

#### 加入团队runner

- 打开apichain，点击右上角可以切换语言

- 点击单机版，即可连接runner。

- 当前版本 选择 联网版

- 服务器 基于您实际运行runner机器的ip地址和端口号、网关等填写，示例中是 http://127.0.0.1:6588。点击 检测 按钮进行网络检测

- 团队 选择 加入团队

- 选择团队 选择 runner开发者
- 点击加入按钮。因您本人是runner的超管，加入团队无需对应团队管理员审批，此时重启apichain后您已加入了runner开发者团队。

#### 探索产品

- 点击顶部您昵称旁边的编辑按钮，修改您在团队内的称呼

- 点击 设置 - 开发环境。这些是您对接口 发送网络请求 以及 执行自动化测试 所需要的 接口地址、环境变量 的数据来源。同一个runner下的不同团队共享同一套开发环境以及该开发环境关联的数据。
- 点击 设置 - 项目。这些是您团队所有负责的微服务的一个列表。同一个runner下的不同团队共享同一个项目列表以及该项目关联的数据。
- 点击 设置 - 版本迭代。这些是您每个开发周期进行版本迭代的一个列表，每个版本迭代包含若干个微服务。可以在版本迭代内记录该迭代所有相关信息，如 各类文档链接、注意事项、上线项目和分支、上线需要做的数据库、配置中心、定时任务的配置等
- 点击 项目 - 文档。会按文件夹分门别类展示该项目的所有接口列表，可以搜索查询，也可以查看该接口的详情，包括请求的path variable、param、header、body，返回的 header、content、cookie等，也可以使用这个接口发送网络请求
- 点击 项目 - 设置。是您的项目在不同开发环境下的数据的一个集合，包括接口地址、数据库信息等。（数据库信息选填，数据库密码会加密存储，主要用于项目编写的接口单测进行结果断言中的数据检查以及清理接口单测产生的脏数据）
- 点击 项目 - 单测。这些测试用例来源于版本迭代开发过程中，对接口的一系列可反复执行的串行调用，以实现某个特定功能的操作流程。可反复执行，依赖于 apichain 提供了各类随机不重复数据的函数，如 随机字符串、当前时间、随机邮箱、随机年龄等。

#### 修改项目数据库连接

项目的数据库连接信息，主要用于项目编写的接口单测进行结果断言中的数据检查以及清理接口单测产生的脏数据。因本项目的单测进行了大量表实际数据变更查询的断言，以及清理单测产生的脏数据，故需要使用数据库配置。目前的数据库配置是错误的，需要您帮忙做修改，以顺利运行项目和迭代内的单测。

- 在项目菜单中找到 runner运行器 这个项目
- 点击 设置
- 选择环境 本地环境
- 域名 修改为 您runner部署的访问地址，域名就是您项目调用的接口地址，通常包括IP地址（域名）和端口号，用 `/`结尾，示例中是 http://192.168.1.5:6588/
- 前缀 是您项目所有接口url地址中公共的那部分路径，此项目中为空
- 数据库主机  根据您项目的数据库实际信息填写，示例中是 192.168.1.5
- 数据库端口  根据您项目的数据库实际信息填写，示例中是 33088
- 数据库用户名  根据您项目的数据库实际信息填写，示例中是 root
- 数据库密码  根据您项目的数据库实际信息填写，示例中是 112233。数据库密码会加密保存，无需担心泄密。
- 数据库名称  根据您项目的数据库实际信息填写，示例中是 apichain
- api请求发送端 实际执行接口调用，发送网络请求的设备，当前设备表示从客户端软件发送网络请求，团队 runner表示从runner发送网络请求，主要用于解决生产环境等高封闭网络无法从外部调用接口发送api请求的问题
- 数据库连接发送端 实际执行sql语句的设备，当前设备表示从客户端软件发送sql语句，团队 runner表示从runner执行sql语句，主要用于解决生产环境等高封闭网络无法从外部执行sql的问题
- 点击最下边 “修改”按钮进行存储。

### 运行单测进行接口校验

- 在项目菜单中找到 runner运行器 这个项目
- 点击单测
- 勾选 入团流程 这个单测用例，选择环境 本地环境，点击右上角 执行用例
- 查看测试用例每个步骤调用的接口、入参和返回
- 点击请求地址可以跳转到相应请求发送的界面，可以重复发送该接口请求
- 查看测试用例每个步骤的一些列断言和对断言的校验
- 勾选 开发环境管理 这个单测用例，重复上述流程
- 勾选 项目测试 这个单测用例，重复上述流程
- 勾选 版本迭代 这个单测用例，重复上述流程
- 勾选 环境变量测试 这个单测用例，重复上述流程
