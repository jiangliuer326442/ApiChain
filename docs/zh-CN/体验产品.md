### 准备一个mysql数据库

您需要准备一个mysql8数据库，如果没有，可参照使用如下脚本启动一个mysql服务

```shell
docker pull mysql:8.0

docker run --name mysql-container -p 33088:3306 -e MYSQL_ROOT_PASSWORD=112233 -d mysql:8.0
```

拥有mysql数据库后，你就可以提供以下环境变量

- DB_HOST

  数据库主机地址，示例中是 192.168.1.5

- DB_PORT

  数据库端口号，示例中是 33088

- DB_USER

  数据库用户名，示例中是 root

- DB_PASS

  数据库密码，示例中是 112233

- DB_NAME

  数据库名称，可以是任意字符串，建议为 apichain

### 运行runner

基于您已经准备好的数据库连接信息，对以下shell命令修改一下数据库连接参数，即可启动一个本地运行的runner

```shell
docker volume create apichain_cache_data;

docker pull registry.cn-shanghai.aliyuncs.com/apichain/runner:1.2.4;

docker run -d 
-p 6588:6588 
-e DB_HOST=192.168.1.5
-e DB_PORT=33088
-e DB_USER=root
-e DB_PASS=112233
-e DB_NAME=apichain
-e DEPLOY_COUNTRY=CN
-e APICHAIN_SUPER_UID=7951db4c0eeecc1f0f399177d9c2d73a
-v apichain_cache_data:/opt/cache
--name apichain-runner
registry.cn-shanghai.aliyuncs.com/apichain/runner:1.2.4
```

### apichain加入团队

- 打开apichain，点击单机版，即可连接runner。

- 当前版本 选择 联网版

- 服务器 基于您实际运行runner的机器的ip地址和端口号、网关等进行填写，示例中是 http://127.0.0.1:6588。点击检测连接按钮

- 团队 选择 加入团队

- 选择团队 选择 runner开发者
- 点击加入按钮。因为您本人是runner的超级管理员，加入团队无需团队管理员审批，此时重启apichain后您已加入了runner开发者团队。

### 修改项目数据库连接

项目的数据库连接信息，主要用于项目编写的接口单测进行结果断言中的数据库检查以及清理接口单测产生的脏数据。

- 在项目菜单中找到 runner运行器 这个项目
- 点击 设置
- 选择环境 本地环境
- 域名 修改为 您runner部署的访问地址，示例中是 http://192.168.1.5:6588/
- 数据库主机  根据您的runner数据库实际信息填写，示例中是 192.168.1.5
- 数据库端口  根据您的runner数据库实际信息填写，示例中是 33088
- 数据库用户名  根据您的runner数据库实际信息填写，示例中是 root
- 数据库密码  根据您的runner数据库实际信息填写，示例中是 112233。数据库密码会加密保存，无需担心泄密。
- 数据库名称  根据您的runner数据库实际信息填写，示例中是 apichain
- 点击最下边 “修改”按钮进行存储。

### 运行单测进行接口校验

- 在项目菜单中找到 runner运行器 这个项目
- 点击单测
- 勾选 入团流程 这个单测用例，选择环境 本地环境，点击右上角 执行用例
- 查看测试用例每个步骤调用的接口、入参和返回
- 点击请求地址可以跳转到相应请求发送的界面，可以重复发送该接口请求
- 查看测试用例每个步骤的一些列断言和对断言的校验
- 勾选 开发环境管理 这个单测用例，重复上述流程
- 勾选 项目测试 这个单测用例，重复上述流程
- 勾选 版本迭代 这个单测用例，重复上述流程
- 勾选 环境变量测试 这个单测用例，重复上述流程
