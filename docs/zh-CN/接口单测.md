# ApiChain：告别“单身狗”式接口测试 🐶

## 😫 之前的痛点

1. **单打独斗**：只能一个个调接口。但现实业务是接力赛，后面的接口眼巴巴等着前面的数据下锅！
2. **只看表面**：Code 码对了就万事大吉？表面 200 OK，数据库里可能早已翻车。

------

## 🚀 ApiChain 硬核解法

### 1. 连招模式：接口串联

把接口装进用例排队打怪！后置接口直接提取前置接口的入参/返回值，数据流转自动成图，告别断层。

### 2. 共享装备：迭代环境变量

某个返回值被疯狂复用？设成迭代环境变量，搭配全局/项目变量，一处定义，到处爽用。

### 3. 透视外挂：直连数据库断言

不轻信表面 Code！支持把步骤数据塞进 SQL，直接去数据库里捞数据比对，真伪一眼看穿。

------

## 🕵️ 数据提取神器：专治花式 JSON/字符串

面对繁杂的 JSON 数组和长串报文，ApiChain 赏你一套提取组合拳：

- `*first()` / `*last()`：**掐头去尾**。快速锁定 JSON 数组的第一个或最后一个对象，继续往下剥洋葱。
- `*random()`：**关联盲盒**。随机取数组元素。*划重点：同一用例内随机种子固定！比如随机抽中用户 A，后续提取的邮箱、用户名都会死死绑定 A，绝不串台。*
- `*length()`：**一把量尺**。直接量出数据长度，断言好帮手。
- `*eval()`：**瑞士军刀**。专切长串字符串！用 JS 代码自定义切割， 代指原始字符串。比如从一长串 URL 里精准抠出 token：`*eval("$$.split('?')[1].split('&')[1].split('=')[1]")`，只有想不到，没有切不出。

------

## 🧹 打完收工：自动清理脏数据

测试跑完数据库一团糟？配个**清理节点**，用自定义 SQL 把脏数据扫地出门，还能随时开关，随心所欲。

------

## 🎁 效率拉满的魔法

- 🎲 **盲盒模式**：入参用随机字符串，同一个用例刷一百遍都不怕冲突！
- 🛡️ **防翻车保险**：用例一键导出给回归测试，专治“改新崩旧”和“一上线就崩”综合征。
- 💾 **一键存档**：前后用例步骤差不多？勾选老用例存为**模板**，新建时直接套用，开工秒速！
